# Contributing to Kairon

Thanks for your interest in contributing! Here's how you can help:

## 🐛 Reporting Issues

- Use the GitHub issue tracker
- Include steps to reproduce, expected vs actual behavior
- Screenshots are always appreciated

## 💡 Feature Requests

Open a discussion or an issue with the `enhancement` label. Tell us:
- What problem you're trying to solve
- How you envision the solution
- Any examples from other projects

## 🔧 Pull Requests

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run `npm run build` to verify
5. Push and open a PR

### Code Style

- TypeScript strict mode enabled
- Follow existing patterns
- Keep things simple — readability > cleverness

## 📖 Documentation

Good docs are as important as good code. If you improved something in the code, update the docs too.

## 🚀 First Time Contributors

Look for issues tagged `good first issue` — we keep some simple ones around for newcomers.
